# dev-workflow

Skills for **developer workflow, Git, GitHub, CLI tooling, and automation**.

## Skills in this category

### Git workflow

| Skill | What it does |
| --- | --- |
| `git-commit` | Write good commit messages — conventional, descriptive, scoped. |
| `git-flow-branch-creator` | Create branches following git-flow naming conventions. |
| `conventional-commit` | Conventional Commits spec — feat, fix, chore, breaking changes. |
| `commit-message-storyteller` | Write narrative commit messages that tell a story. |
| `using-git-worktrees` | Manage multiple working trees with git worktrees. |
| `finishing-a-development-branch` | Wrap up a branch — cleanup, rebase, PR prep. |

### GitHub

| Skill | What it does |
| --- | --- |
| `gh-cli` | GitHub CLI — PRs, issues, releases, actions from the terminal. |
| `github-issues` | Create, triage, and manage GitHub issues. |
| `github-release` | Create GitHub releases — tags, changelogs, assets. |
| `my-issues` | Show your assigned GitHub issues dashboard. |
| `my-pull-requests` | Show your open pull requests dashboard. |
| `pr-dashboard` | Pull request review dashboard — what needs attention. |
| `make-repo-contribution` | Prepare a contribution — fork, branch, PR workflow. |

| Skill | What it does |
