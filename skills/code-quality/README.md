# code-quality

Skills for **code review, refactoring, security audits, and quality assurance**.

## Skills in this category

### Refactoring

| Skill | What it does |
| --- | --- |
| `refactor` | Identify and apply refactoring opportunities — extract, rename, simplify. |
| `refactor-method-complexity-reduce` | Reduce method complexity — extract, decompose, simplify conditionals. |
| `refactor-plan` | Create a refactoring plan before making changes. |
| `review-and-refactor` | Combined code review and refactoring pass. |

### Code review

| Skill | What it does |
| --- | --- |
| `requesting-code-review` | Structure and send effective code review requests. |
| `receiving-code-review` | Process and respond to code review feedback. |
| `quality-playbook` | Run a structured quality playbook — standards, checklists, gates. |
| `doublecheck` | Double-check implementation against requirements. |
| `grill-me` | Adversarial code review — challenge assumptions, find edge cases. |
| `eyeball` | Quick visual code inspection — obvious issues, style, consistency. |
| `roundup` | Aggregate code quality findings into a summary. |
| `roundup-setup` | Set up automated code quality roundup tooling. |

### Security & static analysis

| Skill | What it does |
| --- | --- |
| `security-review` | Security-focused code review — OWASP Top 10, injection, auth. |
| `audit-integrity` | Integrity audit — verify code matches spec, no unintended changes. |
| `codeql` | Run and interpret CodeQL security queries. |

## When to use this category

- "Review this code" / "refactor for clarity"
- "Security review" / "check for OWASP issues"
- "Run CodeQL" / "static analysis"
- "Reduce method complexity"
